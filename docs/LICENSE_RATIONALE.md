# License Rationale

Delphi Commons is provisionally released under the Apache License 2.0.

The project needs an OSI-approved open-source license that supports public auditability, institutional adoption, reproducible research infrastructure, and long-term maintenance. The license governs reuse of the software code only. It does not make study data public and does not authorize violations of participant confidentiality, consent, auditability, human-in-the-loop AI governance, or truthful reporting of Delphi limits.

## License Options Considered

### MIT

MIT is short, permissive, and widely understood. It is easy for universities, nonprofits, public agencies, and commercial organizations to adopt. Its simplicity is useful, but it does not include the same explicit patent grant as Apache 2.0.

### Apache License 2.0

Apache 2.0 is permissive, OSI-approved, and widely accepted for public-interest and institutional software. It includes an express patent grant and clear notice requirements. This is useful for research infrastructure that may be adopted across universities, nonprofits, public agencies, and private organizations.

Apache 2.0 is the provisional default because it balances broad adoption, patent clarity, public auditability, and practical maintainability.

### GPLv3

GPLv3 is a strong copyleft license. It requires derivative works distributed to others to be licensed under GPLv3-compatible terms. This can protect downstream openness, but it may limit adoption by some institutions or vendors that cannot accept strong copyleft obligations.

### AGPLv3

AGPLv3 extends strong copyleft obligations to network services. It is the strongest option for ensuring modified hosted versions remain open. It may be appropriate if maintainers decide that protecting public access to hosted modifications is more important than maximizing institutional or commercial adoption.

## Provisional Recommendation

Use Apache 2.0 unless project maintainers intentionally choose a stronger copyleft model such as GPLv3 or AGPLv3.

Apache 2.0 governs reuse of the original Delphi Commons source code. It does not authorize misuse of the platform, deceptive claims about Delphi outputs, removal of human-in-the-loop safeguards, violation of participant confidentiality, or claims that Delphi consensus establishes correctness.
