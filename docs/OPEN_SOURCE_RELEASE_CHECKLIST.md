# Open Source Release Checklist

## Licensing

- [ ] LICENSE contains full Apache License 2.0 text
- [ ] NOTICE is present
- [ ] README includes license section
- [ ] package metadata uses Apache-2.0
- [ ] source files have appropriate SPDX headers
- [ ] third-party notices reviewed
- [ ] dependency licenses reviewed
- [ ] license rationale documented

## Copyright

- [ ] Copyright owner confirmed
- [ ] Contributor terms documented
- [ ] Third-party code excluded or clearly identified
- [ ] Generated files excluded from copyright headers
- [ ] Unclear-authorship files reviewed

## Governance

- [ ] Ethical Governance Charter included or linked
- [ ] AI Governance & Human-in-the-Loop Contract included or linked
- [ ] Human-subjects limits stated clearly
- [ ] Open-source code does not imply open study data
- [ ] Delphi consensus is not represented as truth
- [ ] AI is not represented as a decision-maker or consensus optimizer

## Security and Privacy

- [ ] SECURITY.md exists
- [ ] Responsible disclosure contact added
- [ ] No secrets committed
- [ ] No participant data committed
- [ ] No identity mappings committed
- [ ] No real consent records committed
- [ ] No sensitive exports committed
- [ ] No local filesystem paths or private Dropbox paths committed
- [ ] Environment files checked
- [ ] Example environment file uses placeholders only

## GitHub Readiness

- [ ] Repository builds locally
- [ ] Tests pass locally
- [ ] README has setup instructions
- [ ] CONTRIBUTING.md exists
- [ ] GOVERNANCE.md exists
- [ ] CITATION.cff exists
- [ ] DCO.md exists if contributor certification will be used
- [ ] Issue templates planned
- [ ] Pull request template planned
- [ ] Code owners planned
- [ ] Initial release version selected

## DOI Readiness

- [ ] Public release created on GitHub
- [ ] Zenodo or equivalent archive connected
- [ ] DOI minted only after release
- [ ] DOI added to README after creation
- [ ] DOI added to CITATION.cff after creation
