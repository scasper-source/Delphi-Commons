# Defect Severity Rubric Template — Phase 1 Dry Run

## Severity definitions
- P0: participant-rights/confidentiality/identity-separation break, auth bypass, severe data exposure.
- P1: major workflow failure or control failure requiring fix before broader rehearsal.
- P2: meaningful usability/workflow defect with workaround.
- P3: minor quality/polish issue.
